## 二、做T降本操作手册

### 核心公式
```
每股利润 = 卖出价 - 买入价
总利润 = 每股利润 × 卖出数量
新均价 = (原持仓成本×原数量 - 卖出价×卖出数量 + 买入价×买入数量) ÷ 总持仓数量
```

### 做T三要素
1. **价差 ≥ 0.5%**：覆盖手续费（佣金万2.5×2 + 印花税0 = 约0.05%）+ 滑点
2. **分批操作**：每次卖1/3仓位，不卖全（防T飞）
3. **高卖低买**：卖出价必须 > 买入价，否则不做

### 做T成本
| 费用 | 金额 |
|------|------|
| 卖出佣金（万2.5） | 约0.025% |
| 买入佣金（万2.5） | 约0.025% |
| 印花税（ETF免） | 0 |
| **合计** | **~0.05%** |

### LOF做T特殊风险：溢价率
- LOF交易价 ≠ 净值，存在**溢价**
- 溢价率 = (交易价 - 净值) ÷ 净值
- 溢价率 > 3%时买入性价比低
- 溢价率 > 10%时，即使标的价格不跌你也亏10%（溢价回归）

## 四、获取实时价格（重要方法论）

### 数据源优先级（2026-06-30修正）
1. **terminal curl 腾讯API（qt.gtimg.cn）** — 首选，最可靠
2. **web_search** 搜索实时价格 — 次选
3. **execute_code urllib 腾讯K线** — 偶尔被环境限制
4. **浏览器访问** — 最慢，超时风险

### 腾讯API直连（最可靠）
```bash
curl -s "https://qt.gtimg.cn/q=sh518880,sz159819,sh562500" \
  -H "User-Agent: Mozilla/5.0" | iconv -f gbk -t utf-8
```

### 价格时效性警告（血泪教训 2026-06-23）
- 搜索结果的价格可能有5-30分钟延迟
- **必须标注"截至XX:XX"或"估算"**
- 如果无法确认最新价格，**告知用户而非编造**
- ⚠️ `web_extract` 从搜狐提取的行情可能是**前一个交易日的盘中数据**
- ⚠️ 用户会当场纠正价格错误 — 每次给出价格前先double-check

## 五、积存金做T

积存金（银行黄金账户）也支持T+0操作：
- 赎回 = 卖出（T+0）
- 买入 = 申购（T+0）
- 手续费：赎回0.5%，买入0.5%（部分银行有优惠）
- 做T方式：高价赎回 → 低价重新买入

### 积存金做T注意
- 价差必须 > 1%才划算（手续费1%）
- 关注**银行赎回价 vs 买入价**的差价
- 工商银行、建设银行、中国银行都支持

## 六、何时不做T+0（重要判断）

以下情况**不做T+0**，等企稳再说：

| 场景 | 判断依据 | 建议 |
|------|----------|------|
| **单日跌幅 > 2%** | 趋势未止，抄底被埋 | 等1-2天企稳 |
| **连续3天下跌** | 恐慌情绪未释放完 | 等放量止跌 |
| **量比 > 2且下跌** | 放量出货 | 不操作 |
| **重大消息面冲击** | 加息/地缘事件 | 等消息消化 |
| **价差 < 0.5%** | 手续费吃掉利润 | 不做 |

### 用户风险偏好适配
- **保守型**（不接受大亏）：只做商品ETF（黄金），不做跨境ETF
- **进取型**（可接受波动）：黄金ETF > 商品ETF > 跨境ETF（需确认T+0）
- **禁止推荐**：债券ETF、货币ETF、国债ETF（用户明确不接受）
